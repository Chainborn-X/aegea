# Tooling

## Tooling Goals

Tools should help the team preserve consistency as the world grows.

Potential tools:

- Asset naming validators.
- Prompt library generators.
- Region checklists.
- Map metadata validators.
- Lore and terminology indexes.
- Export scripts for tiles, sprites, concepts, and audio.
- Build scripts and release packaging.

## Principles

- Automate repeated production checks.
- Keep tools documented.
- Prefer simple scripts until the workflow proves it needs more.
- Make validation rules reflect the creative direction where practical.

## Future Tooling Ideas

- Scene checklist: verifies each area has a discovery hook, landmark, palette tag, biome tag, and traversal notes.
- Asset manifest: tracks concept, source, export, license, and implementation status.
- Prompt index: stores reusable prompts and negative prompt language by category.
