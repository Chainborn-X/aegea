# Aegea

Aegea is a top-down action RPG set in a forgotten mythological Mediterranean world inspired by ancient Greece.

This repository is the long-term source of truth for the complete game project: creative direction, worldbuilding, game design, technical decisions, source code, artwork, music, level designs, lore, tools, and production assets.

The goal is not simply to build a game. The goal is to create a world players long to return to.

## Vision

Aegea is inspired by the sense of wonder and freedom found in games such as:

- Battle of Olympus
- Secret of Mana
- Chrono Trigger
- The Legend of Zelda

The player begins alone near the sea and is free to explore a vast world filled with ancient ruins, mysterious caves, sacred temples, forgotten gods, hidden pathways, and distant horizons.

The emotional core of the project is:

> I am alone. The world is vast. Something incredible is waiting beyond the next hill.

Every design decision should reinforce that feeling.

## Emotional Pillars

Aegea should evoke:

- Wonder
- Discovery
- Freedom
- Solitude
- Adventure
- Mystery
- Hope

Aegea should avoid:

- Grimdark themes
- Excessive cynicism
- Generic medieval fantasy
- Heavy anime influences
- Cartoonish presentation
- Theme park world design

## How To Use This Repository

- Use `docs/vision` for the emotional foundation and non-negotiable creative goals.
- Use `docs/world` for lore, regions, mythology, creatures, factions, and continuity.
- Use `docs/art` for visual direction, palettes, environments, architecture, characters, creatures, and UI.
- Use `docs/design` for game mechanics, exploration, combat, progression, items, and quests.
- Use `docs/technical` for engine decisions, rendering notes, build pipelines, and tooling.
- Use `assets` for reference material and production assets, organized by discipline.
- Use `prompts` for reusable AI art and design prompts that preserve the Aegea identity.
- Use `prototypes` for experiments that may or may not become part of the final game.
- Use `game` for the eventual playable game source.
- Use `tools` for scripts, exporters, validators, importers, and production utilities.

## Consistency Matters

Every new asset, feature, quest, tool, and document should be checked against the emotional core and the art direction. Aegea is not merely a collection of Greek-inspired content. It is a specific feeling: a small traveler crossing a sunlit, ancient, mysterious world.

When adding new work, ask:

- Does this make the world feel vast?
- Does it invite discovery?
- Does it preserve solitude without becoming bleak?
- Does it feel mythological rather than historical?
- Does it avoid generic medieval fantasy?
- Does it make players wonder what lies beyond the visible edge of the scene?

## Current Status

This repository begins as a foundation for vision, worldbuilding, art direction, design, prompt libraries, and future technical work. Production code and assets should grow from these documents rather than replacing them.
